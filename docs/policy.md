# 정책 (공유 비즈니스 규칙)

여러 기능이 공유하는 규칙의 **진실의 원천**입니다. 기능별 문서에 중복하지 말고 여기를 참조하세요.

## 공통 규칙

- **시간/날짜**: 저장은 UTC 기준, 표현 계층에서 변환. 기간은 `end`가 `start`보다 뒤여야 함.
- **감사(audit)**: 영속 엔티티는 `createdAt` / `updatedAt`을 자동 관리(JPA Auditing).
  설정은 `config/JpaAuditingConfig`로 분리(코드 컨벤션 참조).
- **식별자**: PK 전략은 테이블 명세를 따른다(기본 IDENTITY).
- **에러 코드 네임스페이스**: 대문자 SNAKE_CASE. 검증 실패는 공통 `VALIDATION_ERROR`,
  도메인 규칙 위반은 규칙별 코드.

## 추가 규칙 등록 방법

새 공유 규칙이 생기면 여기에 항목을 추가하고, 관련 기능 문서에서는 링크로만 참조합니다.
한 규칙이 두 곳에 다르게 적히면 이 문서가 우선합니다.
